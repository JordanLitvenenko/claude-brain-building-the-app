# Session Management Guidelines

## Token Management
### Monitoring Approach
- User will monitor token usage
- Start new session when approaching limits
- Save critical work before limits

### Session Transitions
1. Complete current documentation updates
2. Verify all valuable information is stored
3. End current session cleanly
4. Start fresh session
5. Reference previous session's documentation for context

### Continuity Management
- Ensure brain structure maintains context between sessions
- Use daily logs for session continuity
- Reference previous session endpoints
- Maintain progress tracking across sessions

## Best Practices
### Session Length
- Monitor for token consumption
- Plan natural break points
- Complete documentation before breaks
- Ensure clean session transitions

### Information Preservation
- Priority on storing valuable insights
- Complete current documentation tasks
- Verify brain updates before breaks
- Maintain context across sessions

### Transition Points
- Recognize approaching limits
- Plan orderly transitions
- Complete current thought/task
- Document transition points

## AI Responsibilities
- Complete current documentation
- Store valuable information
- Maintain session context
- Prepare for next session
- Track progress across sessions
